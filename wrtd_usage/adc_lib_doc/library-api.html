struct adc_dev *adc_open(char *name, unsigned int dev_id, unsigned long totalsamples, unsigned int nbuffer, unsigned long flags)¶

It opens a link with an ADC device

Return
It returns an opaque object describing the device. It should be passed around but not be looked into.
Parameters
  • name: name of the board type to open according to adc_board_name.
  • dev_id: The device identifier is used to enumerate several cards in the same system. The number is usually dependent on the geographic placement of the card (bus number, slot number).
  • totalsamples: This is a a hint about how big a buffer the application will use. i.e. nshots * (presamples + postsamples)
  • nbuffer: This is a hint about how many buffers are being used at the same time by the application. For example, a multi-shot acquisition requires all buffers to be available at the same time. Again, this information may require device-level configuration at open time.
  • flags: currently it supports: ADC_F_FLUSH, ADC_F_VERBOSE

It initialize the board, this can be different board to board.

int adc_acq_start(struct adc_dev *dev, unsigned int flags, struct timeval *timeout)¶

It tells hardware to start the acquisition on the given device according to the current configuration (but starting in itself is usually a fast operation, because it doesn’t rewrite configuration to the hardware, at least with currently supported boards). It can return immediately or wait for completion, with a timeout.

Return
0 on success, -1 on error and errno is set appropriately ADC_EDISABLED - acquisition disabled or aborted
Parameters
  • dev: ADC device token
  • flags: flags to control the acquisition options ADC_F_FLUSH - discard prevoius leftover data before activating the acquisition (if any is there)
  • timeout: The timeout is used like the argument for select(2): if NULL, the functions wait forever, otherwise they specify the maximum allowed waiting time and passing zero values asks to not wait at all (you can use adc_acq_poll() later).

int adc_fill_buffer(struct adc_dev *dev, struct adc_buffer *buf, unsigned int flags, struct timeval *timeout)¶

It ask the library to get the data from the device and fill the given buffer.

The size of the buffer must be at least equal to the size of the shot that library should copy into; otherwise, the result is undefined.

Return
0 on success, -1 on error and errno is set appropriately ADC_EDISABLED - when the acquisition is disabled or aborted ETIMEDOUT - timeout
Parameters
  • dev: ADC device token
  • buf: the buffer to fill
  • flags: used to control how to fill the buffer (ADC_F_FIXUP)
  • timeout: maximum time to fill the buffer. The behavior is similar to select(1). If NULL, there is no timeout and the function will wait until the data is ready.

struct adc_timestamp *adc_tstamp_buffer(struct adc_buffer *buf, struct adc_timestamp *ts)¶

It gets the timestamp of a given buffer, in a driver-specific way (most likely looking in the metadata structure).

If the caller passes a

ts pointer, the timestamp is copied into that pointer and the same pointer is returned. If ts is NULL, the function returns a pointer of its own choice, that is only valid until the buffer is released.
Return
the buffer timestamp
Parameters
  • buf: the buffer of interest
  • ts: the buffer timestamp
